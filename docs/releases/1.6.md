# NativeGallery 1.6

**Дата:** 2026-07-02  
**Тег:** `release-1.6`  
**Релиз:** [GitHub Releases](https://github.com/Lanmikeman/nativegallery/releases/tag/release-1.6)  
**Репозиторий:** [Lanmikeman/nativegallery](https://github.com/Lanmikeman/nativegallery)

Актуальное состояние ветки `main`: OpenVK, фотоконкурсы, личный кабинет, роль владельца сервера, страницы помощи и отчётов конкурса. Заменяет промежуточный тег `release-1.5`.

## Основное

### Авторизация и админка

- **OpenVK** — вход, регистрация, привязка аккаунта (`/lk/profile?type=OpenVK`)
- Управление инстансами OpenVK через overlay `storage/auth-settings.json` (`/admin?type=AuthSettings`)
- **Редактирование новостей** — создание, правка с пометкой «отредактировано», удаление (`/admin?type=News`)
- **Права пользователей** — `/admin?type=UserEdit&user_id=`
- **Роль владельца (`admin = 4`)** — раздел «Сервер» в админке:
  - переключатель Debug (Tracy);
  - редактирование параметров `root` через `storage/server-settings.json` (базовый `ngallery.yaml` не трогается)

### Фотоконкурсы

- Голосование: `/voting`, претенденты `/voting/waiting`, победители `/voting/results`
- Рейтинг авторов-победителей: `/voting/rating`
- Подача фото: `/voting/sendpretend` (требуется вход)
- Отчёт по конкурсу для фото-победителя: `/pk.php?pid={id}&type=d` (алиас `/voting/report`)
- Cron: `deploy/setup-cron.sh` → `ExecContests.php` каждые 5 минут
- Админка: создание, принудительное завершение и отмена (`/admin?type=Contests`)

### Личный кабинет и публичные страницы

- `/lk/ticket.php` — мои заявки на изменение БД (legacy URL)
- `/lk/konkurs.php` — участие в фотоконкурсе (legacy URL)
- `/help/` — страница помощи (правила, конкурс, ссылки)
- Слоган подвала из `footerslogan` в конфиге

## Обновление с 1.4

```bash
cd /var/www/nativegallery   # или любой NG_WEB_ROOT (пример: /mnt/win/nativegallery)
git fetch --tags
git checkout release-1.6    # или git pull origin main
composer install --no-dev

# только если ещё не применяли:
mysql -u USER -p DATABASE < sqlcore/sql_0008.sql

mkdir -p storage/locks
sudo chown -R www-data:www-data storage logs
sudo chmod -R 775 storage logs

# cron для конкурсов (если ещё не установлен):
sudo NG_WEB_ROOT=/path/to/nativegallery bash deploy/setup-cron.sh
sudo -u www-data php app/Controllers/Exec/Tasks/ExecContests.php

sudo systemctl reload nginx php8.3-fpm
```

Миграции `sql_0005` … `sql_0007` нужны только при обновлении с версии ≤ 1.3.

## Первый владелец сервера

Роль владельца не требует SQL-миграции — достаточно обновить поле в БД:

```sql
UPDATE users SET admin = 4 WHERE id = ВАШ_ID;
-- или по нику:
UPDATE users SET admin = 4 WHERE username = 'ваш_ник';
```

Раздел **Админка → Сервер** (`/admin?type=ServerSettings`) появится только у пользователя с `admin = 4`.

## Legacy-заглушки для Nginx

При развёртывании на Nginx без полного rewrite для `.php` URL убедитесь, что в репозитории присутствуют файлы-заглушки (они перенаправляют на front controller):

| Файл | Назначение |
|------|------------|
| `pk.php` | `/pk.php?pid=…&type=d` — отчёт конкурса |
| `help/index.php` | `/help/` — страница помощи |
| `lk/ticket.php` | Заявки БД в личном кабинете |
| `lk/konkurs.php` | Конкурс в личном кабинете |

## Уровни доступа

| `users.admin` | Роль |
|---------------|------|
| `0` | Пользователь |
| `1` | Администратор |
| `2` | Фотомодератор |
| `3` | Модератор |
| `4` | Владелец сервера |
