# NativeGallery 1.8

**Дата:** 2026-07-05  
**Ветка:** `main` (тег `release-1.8` — после публикации на GitHub)  
**Репозиторий:** [Lanmikeman/nativegallery](https://github.com/Lanmikeman/nativegallery)

Откат экспериментальных функций к стабильной схеме **upstream (claradex)**: комментарии к фото, полная перезагрузка страниц. Полное удаление музыкального раздела, добавленного в релизе 1.7.

## Удалено

### Музыкальный раздел (был в 1.7)

| Что убрано | Подробности |
|------------|-------------|
| Страница `/music` | Личная библиотека, M3U, плейлисты |
| Мини-плеер в шапке | `MusicPlayer.php`, `music-player.js`, `music-player.css` |
| API `/api/audio/*` | upload, stream, playlist, proxy, metadata, library, delete |
| Админка | `MusicSettings`, `RadioStations`, `/api/admin/settings/music`, `/api/admin/radio/*` |
| Сервис | `AudioLibrary.php`, контроллеры `Api/Audio/*` |
| Миграции | `sql_0010.sql`, `sql_0011.sql` удалены из репозитория |
| Конфиг | Секция `root.audio` в `ngallery-example.yaml` и дефолтном Docker-конфиге |
| Прочее | BBCode `[music]`, смайлы `music.gif`/`music2.gif`, `playerjs.js`, кнопка «Музыка» в меню |

Если таблицы уже созданы на сервере, выполните очистку:

```bash
mysql -u USER -p DATABASE < sqlcore/drop_legacy_tables.sql
```

В `storage/auth-settings.json` можно вручную удалить блок `"audio": { ... }` — на работу сайта это не влияет.

### SPA-навигация (эксперимент из 1.7)

- `routing.js` **закомментирован** в `LoadHead.php` — переходы по меню снова с **полной перезагрузкой** страницы, как в [claradex/nativegallery](https://github.com/claradex/nativegallery).
- Файл `static/js/routing.js` остаётся в репозитории, но **не подключается** по умолчанию.

### Социальная база и личные сообщения (эксперимент)

Если применялись миграции после коммита `7bd5fb2`:

```bash
mysql -u USER -p DATABASE < sqlcore/rollback_after_7bd5fb2.sql
```

Удаляются таблицы: `message_*`, `social_*`, `user_friendships`, `user_identity_links`.

## Изменено

### Комментарии к фотографиям

Восстановлена схема **origin**:

- `comments.js` грузится **внизу** `Photo.php` с `defer`, не в `<head>`.
- Форма: `type="submit"`, `action="/comment.php"`.
- Inline-обработчик в `Photo.php`: `$('#f1').submit()` → `POST /api/photo/comment`.
- Исправлен `Create.php`: проверка `hasUploadedFile()` перед `finfo_file()` (нет 500 на пустом вложении).

### Прочее

- Синяя звезда владельца сервера (`UserRoleBadge`) — как у админа.
- Legacy API: `/api/photo/loadnew`, заглушка `get-pub-photos` в `api.php`.

## Обновление с 1.7

```bash
cd /var/www/nativegallery   # или ваш NG_WEB_ROOT
git pull origin main
composer install --no-dev

# если ставили музыку (sql_0010 / sql_0011):
mysql -u USER -p DATABASE < sqlcore/drop_legacy_tables.sql

# если ставили соцбазу / сообщения:
# mysql -u USER -p DATABASE < sqlcore/rollback_after_7bd5fb2.sql

sudo systemctl reload nginx php8.3-fpm
```

В браузере: **Ctrl+Shift+R** (жёсткое обновление без кэша JS).

После обновления при отправке комментария на `/photo/ID/` в Network должен появиться `POST /api/photo/comment`.
