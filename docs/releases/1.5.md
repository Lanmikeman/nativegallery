# NativeGallery 1.5

**Дата:** 2026-07-02  
**Тег:** `release-1.5`  
**Репозиторий:** [Lanmikeman/nativegallery](https://github.com/Lanmikeman/nativegallery)

Дополнения к релизу 1.4: OpenVK-авторизация, редактирование новостей, доработка админ-панели.

## Основное

- **OpenVK** — вход через [openvk.org](https://openvk.org) и [vepurovk.xyz](https://vepurovk.xyz), привязка аккаунта в `/lk/profile?type=OpenVK`
- **Редактирование новостей** — админка `/admin?type=News`, пометка «отредактировано» на `/news2` и главной
- **Админка** — страницы Galleries и EntityEdit, исправлена маршрутизация по `?type=`
- **Исправления** — пустая галерея `/article/{id}`, предупреждения PHP 8.3

## Обновление с 1.4

```bash
git pull
composer install --no-dev

mysql -u USER -p DATABASE < sqlcore/sql_0008.sql

# при необходимости настройте OpenVK в ngallery.yaml:
# openvk.enabled: true
# openvk.redirect_uri: 'https://ВАШ_ДОМЕН/auth/callback'

sudo systemctl reload nginx php8.3-fpm
```

## Настройка OpenVK

1. В `ngallery.yaml` включите `openvk.enabled: true`.
2. Укажите `redirect_uri` (или оставьте пустым — подставится `https://ВАШ_ДОМЕН/auth/callback`).
3. На стороне OpenVK зарегистрируйте приложение с этим callback URL.
4. Для openvk.org обязательно `api_domain: https://api.openvk.org`.

Подробнее: [configuration.md](../configuration.md#вход-через-openvk).
